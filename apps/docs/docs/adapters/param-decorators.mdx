# Param Decorators

Param decorators let you extract specific values from the execution context directly in handler methods, keeping them clean and free of boilerplate. They work on `execute()` and on any `@Autocomplete()` method.

## `@Ctx()`

Injects the raw `ExecutionContext`, which wraps the underlying interaction and exposes its name and type.

```ts
execute(@Ctx() ctx: ExecutionContext) {
  console.log(ctx.name); // 'ping'
}
```

## `@Guild()`

Injects the `Guild` object from the interaction. Returns `null` if the command was used in a DM.

```ts
execute(@Guild() guild: Guild | null) {
  return guild ? guild.name : 'DM';
}
```

## `@Author()`

Injects the `User` who triggered the interaction.

```ts
execute(@Author() user: User) {
  return `Hello, ${user.username}!`;
}
```

## Combining decorators

You can use multiple param decorators in a single `execute()` method:

```ts
execute(@Author() user: User, @Guild() guild: Guild | null): string {
  return `${user.username} in ${guild?.name ?? 'DMs'}`;
}
```

## `@Option(name?)`

Injects the value of a specific slash command option by name. Returns `string | number | boolean | undefined`.

```ts
execute(@Option('query') query: string, @Option('count') count: number) {
    return `Searching for "${query}" (${count} results)`;
}
```

Call `@Option()` without arguments to receive all options at once as a `Record<string, string | number | boolean>`:

```ts
execute(@Option() options: Record<string, string | number | boolean>) {
    console.log(options.query, options.count);
}
```

For typed options that resolve to Discord objects (User, Channel, Role), use `@Ctx()` and access them via the raw interaction.

## Combining decorators

You can mix and match any param decorators in a single method:

```ts
execute(
    @Author() user: User,
    @Option('query') query: string,
    @Guild() guild: Guild | null,
): string {
    return `${user.username} searched "${query}" in ${guild?.name ?? 'DMs'}`;
}
```

## Usage in autocomplete methods

All param decorators work the same way on `@Autocomplete()` methods. This keeps the contract consistent across the entire handler class:

```ts
@Autocomplete('query')
async autocompleteQuery(@Option('query') focused: string, @Author() user: User) {
    return db.search(focused, user.id).map((r) => ({ name: r.title, value: r.id }));
}
```

## How they work

Param decorators are resolved by the `CommandExecutor` via a registry of `paramResolvers`. Each resolver receives the `ExecutionContext` and an optional `data` payload (e.g. the option name) and returns the value to inject at that parameter position. The discord.js adapter registers resolvers for `CONTEXT`, `GUILD`, `AUTHOR`, and `OPTION` on initialization.
